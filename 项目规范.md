# AutoQuant Research Agent BigQuant 比赛项目开发规范

你是量化研究工程师和 Python 工程师。请按照以下规范，为 BigQuant 比赛项目编写代码。

## 1. 项目定位

项目名称：

AutoQuant Research Agent：基于 LLM Agent 的量化策略研发与回测诊断系统

项目目标：

用户输入自然语言量化研究想法，系统自动完成：

1. 策略假设结构化
2. 候选因子生成
3. BigQuant 数据获取
4. 因子计算
5. IC 分析
6. 分层收益分析
7. Top 10% 等权组合简化回测
8. 风险指标计算
9. 策略诊断
10. Markdown 研究报告生成

项目不是荐股助手，不输出具体股票买卖建议。所有输出只用于历史回测、统计检验、策略研究和风险诊断。

## 7. 字段探测必须先做

在正式写 load_data_bigquant() 前，必须先写字段探测函数。

目标：

1. 检查表是否有权限访问
2. 打印字段名
3. 保存字段清单
4. 根据字段自动判断使用 turn 还是 turnover
5. 根据字段自动判断使用 st_status 还是 is_risk_warning

示例代码：

```python
from bigquant import dai
import pandas as pd
from pathlib import Path

def probe_table(table_name: str, limit: int = 5) -> pd.DataFrame:
    sql = f"""
    SELECT *
    FROM {table_name}
    ORDER BY date, instrument
    LIMIT {limit}
    """
    df = dai.query(sql).df()
    print(f"\n===== {table_name} =====")
    print("shape:", df.shape)
    print("columns:")
    print(list(df.columns))
    display(df.head())
    return df

def run_data_probe():
    tables = [
        "cn_stock_bar1d",
        "cn_stock_prefactors_community",
        "cn_stock_prefactors",
        "cn_stock_factors_base",
        "cn_stock_status",
        "cn_stock_index_bar1d",
    ]

    results = {}
    for table in tables:
        try:
            results[table] = probe_table(table)
        except Exception as e:
            print(f"[WARN] failed to probe {table}: {e}")

    return results
```

Notebook 第一阶段必须运行：

```python
probe_results = run_data_probe()
```

如果某张表不存在或无权限，不要让程序崩溃，要降级使用可用表。

## 10. 未来函数和数据泄露规则

这是最重要的代码约束。

### 10.1 特征列规则

特征只能使用当前日和过去数据：

允许：

```sql
m_lag(close, 5)
m_avg(volume, 5)
m_std(close, 20)
close / m_lag(close, 5) - 1
volume / m_avg(volume, 5)
```

禁止在特征中使用：

```sql
m_lead(...)
future_return_*
未来收益
未来价格
未来成交量
```

### 10.2 标签列规则

未来收益标签必须命名为：

```text
future_return_1d
future_return_3d
future_return_5d
```

标签可以用：

```sql
m_lead(close, 1) / close - 1 AS future_return_1d
m_lead(close, 3) / close - 1 AS future_return_3d
m_lead(close, 5) / close - 1 AS future_return_5d
```

但标签只能用于评价，不能用于选股排序。

### 10.3 自动校验

必须写一个函数检查未来函数泄露：

```python
def validate_no_future_leakage(feature_cols):
    bad_cols = [c for c in feature_cols if c.startswith("future_")]
    if bad_cols:
        raise ValueError(f"Feature columns contain future labels: {bad_cols}")
```

策略排序时只能使用 factor 列，不能使用 future_return 列。

## 11. MVP 因子设计

第一版只做量价因子。

用户策略想法：

```text
研究量比放大、换手率上升但股价还没有明显上涨的股票，未来是否有超额收益。
```

候选因子：

### 11.1 volume_ratio_5d

定义：

```text
当日成交量 / 过去 5 日平均成交量
```

SQL：

```sql
volume / m_avg(volume, 5) AS volume_ratio_5d
```

### 11.2 turnover_ratio_5d

定义：

```text
当日换手率 / 过去 5 日平均换手率
```

如果换手率字段为 turn：

```sql
turn / m_avg(turn, 5) AS turnover_ratio_5d
```

如果换手率字段为 turnover：

```sql
turnover / m_avg(turnover, 5) AS turnover_ratio_5d
```

### 11.3 price_reaction_lag

定义：

```text
成交量放大，但当日价格涨幅不大。
```

示例：

```sql
volume / m_avg(volume, 5) - abs(close / m_lag(close, 1) - 1) AS price_reaction_lag
```

### 11.4 volume_price_divergence

定义：

```text
成交量变化与价格变化的背离。
```

示例：

```sql
(volume / m_avg(volume, 5)) / (abs(close / m_lag(close, 1) - 1) + 0.001) AS volume_price_divergence
```

### 11.5 return_5d

定义：

```text
过去 5 日收益率。
```

SQL：

```sql
close / m_lag(close, 5) - 1 AS return_5d
```

## 12. 推荐数据获取函数

注意：字段名可能不同。正式代码必须根据 probe 结果自动选择 turnover 字段。

```python
from bigquant import dai
import pandas as pd

def load_data_bigquant(
    start_date: str = "2020-01-01",
    end_date: str = "2024-12-31",
    table_name: str = "cn_stock_prefactors_community",
    turnover_col: str = "turn",
) -> pd.DataFrame:
    sql = f"""
    SELECT
        date,
        instrument,
        close,
        volume,
        {turnover_col} AS turnover,
        close / m_lag(close, 5) - 1 AS return_5d,
        volume / m_avg(volume, 5) AS volume_ratio_5d,
        {turnover_col} / m_avg({turnover_col}, 5) AS turnover_ratio_5d,
        abs(close / m_lag(close, 1) - 1) AS abs_return_1d,
        volume / m_avg(volume, 5) - abs(close / m_lag(close, 1) - 1) AS price_reaction_lag,
        (volume / m_avg(volume, 5)) / (abs(close / m_lag(close, 1) - 1) + 0.001) AS volume_price_divergence,
        m_lead(close, 1) / close - 1 AS future_return_1d,
        m_lead(close, 3) / close - 1 AS future_return_3d,
        m_lead(close, 5) / close - 1 AS future_return_5d
    FROM {table_name}
    WHERE
        date >= '{start_date}'
        AND date <= '{end_date}'
    QUALIFY
        COLUMNS(*) IS NOT NULL
    ORDER BY date, instrument
    """
    df = dai.query(sql).df()
    df["date"] = pd.to_datetime(df["date"])
    return df
```

如果 `QUALIFY COLUMNS(*) IS NOT NULL` 在当前环境不兼容，则改为 pandas 后处理：

```python
df = df.dropna()
```

IC 用 Spearman 相关：

```python
def calc_daily_ic(df, factor_col, target_col):
    ic = (
        df.groupby("date")
        .apply(lambda x: x[factor_col].corr(x[target_col], method="spearman"))
        .dropna()
    )
    return ic
```

分层收益：

```python
def calc_quantile_return(df, factor_col, target_col, n_quantiles=5):
    data = df[["date", "instrument", factor_col, target_col]].dropna().copy()

    def assign_quantile(x):
        try:
            return pd.qcut(x, n_quantiles, labels=False, duplicates="drop") + 1
        except Exception:
            return pd.Series(index=x.index, data=np.nan)

    data["quantile"] = data.groupby("date")[factor_col].transform(assign_quantile)
    result = data.groupby("quantile")[target_col].mean()
    return result
```

def main():

import matplotlib.pyplot as plt
from pathlib import Path

def plot_cumulative_return(daily_returns, path="outputs/cumulative_return.png"):
    Path("outputs").mkdir(exist_ok=True)
    cumret = (1 + daily_returns).cumprod() - 1
    plt.figure(figsize=(10, 5))
    plt.plot(cumret.index, cumret.values)
    plt.title("Cumulative Return")
    plt.xlabel("Date")
    plt.ylabel("Return")
    plt.grid(True)
    plt.savefig(path, dpi=150, bbox_inches="tight")
    plt.close()
