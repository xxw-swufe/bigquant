打开 Notebook → Kernel → **Restart Kernel**（让代码改动生效）→ 顺序跑 cell。

## 第二层：端到端 smoke（不进 Notebook，但用真实数据）

```bash
cd /Users/yangpeng/Desktop/ai量化
.venv-vscode/bin/python -c "
import sys, os
sys.path.insert(0, '.')
from dataclasses import replace
from src.config import DEFAULT_CONFIG
from src.pipeline import run_research_pipeline

config = replace(DEFAULT_CONFIG, data_backend='local',
                 local_etf_parquet='data/parquet/local_etf_daily.parquet')

for q in ['研究成交额放大超过1.5倍、当日上涨但涨幅小于2%的ETF，隔日上涨概率是否更高？',
          '研究成交额放大、20日动量强、波动率低的ETF，未来5日是否有超额收益？']:
    r = run_research_pipeline(q, config=config, use_llm=False, outputs_dir='outputs/smoke')
    print(f'{r[\"task_type\"]:25s} | {len(r.get(\"plot_paths\", []))} 张图 | {r[\"report_path\"]}')
"
```

**合格标准**：
- 第 1 个 query：`task_type=condition_research`，3 张图
- 第 2 个 query：`task_type=factor_research`，≥10 张图
- 两个 report 都落到 `outputs/smoke/`

## 第三层：Notebook 跑 cell 4（最直观）

### 3.1 准备
1. 重启 Kernel
2. 跑 cell 1（setup）
3. **在 cell 4 顶部手动设 `user_idea`**（因为 cell 4 故意不写死默认问题了）：

```python
user_idea = "缩量上涨后下一日怎么样"
```

### 3.2 跑 cell 4

**预期**：
- 任务类型：`condition_research`
- 报告路径：`outputs/condition_research_report_<时间戳>.md`
- 图表数量：3 张（probability_bar, yearly_stability, instrument_stability）
- Cell 下方**直接渲染** Markdown 报告 + 3 张图

### 3.3 测多因子（核心场景）

把 cell 4 顶部 `user_idea` 改成：

```python
user_idea = "研究20日动量和低波动ETF未来5日表现"
```

**预期**：
- 任务类型：`factor_research`
- 图表数量：≥10 张
- 报告下方直接渲染 Markdown + 所有图

### 3.4 测 LLM 路由（可选，需要 API key）

跑 cell 1 → cell 3（chat UI 控件），然后**在控件里输入**：

| 输入 | 预期 task_type | 预期图表 |
|------|---------------|---------|
| `研究成交额放大、20日动量强、波动率低的ETF，未来5日是否有超额收益？` | `factor_research` | 10+ |
| `研究成交额放大超过1.5倍、当日上涨但涨幅小于2%的ETF，隔日上涨概率是否更高？` | `condition_research` | 3 |
| `研究连续3天下跌后，ETF 次日反弹概率是否提高？` | `condition_research` | 3 |
| `研究BBI指标选ETF` | `direct_reply` | 0，落盘 `unmet_research_plan_*.md` |
| `换手率加量比加5日成交额的ETF，未来5日是否上涨？` | `direct_reply` | 0，落盘 `unmet_research_plan_*.md` |
| `你好` | `direct_reply` | 0，**不落盘**（闲聊） |

> **注意**：cell 3 的 chat UI 5 秒内同样文本不会重复发送（防止双击误触）。
> 想强制重发，先**修改输入框的文本**，再点发送。
