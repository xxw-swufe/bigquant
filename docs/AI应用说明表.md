# AI 应用说明表

| 模块 | 是否 AI 核心 | 说明 |
| --- | --- | --- |
| 条件研究解析 | 是 | 将用户自然语言条件问题转成结构化事件规则 |
| 条件研究诊断 | 是 | 诊断条件样本数量、概率提升、收益提升和稳定性 |
| 条件研究报告 | 是 | 自动生成条件事件研究 Markdown 报告 |
| 策略理解 | 是 | 后续扩展：将用户自然语言 ETF 研究想法转成结构化研究假设 |
| 因子生成 | 是 | 后续扩展：根据研究假设生成候选 ETF 因子，并标记经济含义和未来数据风险 |
| 表达知识库路由 | 是 | 将“今日上涨”“缩量上涨”“量价分”“趋势走强”等口语表达映射到标准术语、条件、因子和目标 |
| 数据探测 | 否 | 探测 BigQuant 表字段和权限 |
| 数据加载 | 否 | 从 BigQuant 加载 ETF 日频数据和标签 |
| 条件事件分析 | 否 | 计算事件样本、基准样本、上涨概率、概率提升和分年份稳定性 |
| 单因子分析 | 否 | 计算 IC、ICIR、分层收益和覆盖率 |
| 因子标准化 | 否 | 横截面 rank 标准化并统一因子方向 |
| 综合评分 | 否 | 根据权重计算 ETF 综合得分 |
| 简化回测 | 否 | 按综合得分选择 Top 10% ETF 等权持有 |
| 策略诊断 | 是 | 诊断单因子有效性、组合风险和权重合理性 |
| 报告生成 | 是 | 自动生成 Markdown 研究报告和免责声明 |

## AI 核心标注

代码中以下函数使用 `# [AI-CORE]` 标注：

- `parse_user_strategy`
- `classify_expression`
- `parse_condition_research`
- `generate_factor_candidates`
- `diagnose_condition_result`
- `generate_condition_report`
- `diagnose_strategy`
- `generate_report`

## 合规边界

系统只输出历史研究结论和风险诊断，不输出具体证券买卖建议，不预测未来确定性涨跌。
