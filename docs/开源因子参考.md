## 4. 因子大类

本项目按股市常用因子体系整理为 8 大类：

正式因子库分四层维护：

- 原始数据层：回答“我有什么数据”，包括 `OHLCV`、`AMO`、换手率、指数涨跌、财务数据、资金流字段、ETF 专属字段。
- 同花顺原生指标层：回答“用户说的指标是什么”，记录用户看盘时会直接说出的指标名称，例如 `MA`、`MACD`、`KDJ`、`RSI`、`BOLL`、`OBV`、`BIAS`、`DMI`、`SAR`、`VR`、量比、换手率、主力净流入、阶段涨幅、阶段新高。
- 加工因子层：回答“这个指标怎么变成可检验的因子”，把原始数据或原生指标加工成可检验的因子或条件，例如 `amount_ratio_20d`、`momentum_20d`、`ma_gap_60d`、`obv_divergence_20d`、`price_volume_breakout_20d`。
- 综合评分层：回答“最终怎么形成研究结论或打分”，把多个加工因子合成为大类评分，例如量价分、动量分、趋势分、反转分、风险分、估值分、质量分、成长分。

常用周期统一为 `3日`、`5日`、`10日`、`20日`、`60日`、`120日`、`250日`。因子库可以完整记录多个周期，但最终打分不能把高度重复的周期全部塞进去，应根据研究目标选择少数代表周期。

### 4.1 量价因子

回答：有没有资金进来？上涨是不是有量？下跌是不是放量杀？有没有放量滞涨、缩量回调、量价背离？

### 4.1.1 成交额/成交量活跃度

- `amount_ratio_5d`、`amount_ratio_10d`、`amount_ratio_20d`、`amount_ratio_60d` — 今日成交额 / N日平均成交额；同花顺看 `AMO`、成交额、成交额均线、`AMO / MA(AMO,N)`。
- `volume_ratio_5d`、`volume_ratio_10d`、`volume_ratio_20d`、`volume_ratio_60d` — 今日成交量 / N日平均成交量；同花顺看 `VOL`、成交量、均量线、`VOL / MA(VOL,N)`。
- `volume_ratio_intraday` — 盘中量比；同花顺直接看“量比”，用于判断盘中成交是否突然活跃。
- `turnover_rate` — 当日换手率；同花顺直接看“换手率”，用于判断筹码交换是否活跃。
- `turnover_ratio_5d`、`turnover_ratio_20d` — 今日换手率 / N日平均换手率，用于识别突然放大的筹码交换。

ETF 优先看成交额，其次看成交量。成交额更能反映真实资金规模，成交量更多用于辅助确认。

### 4.1.2 量价确认

- `up_volume_confirm_5d`、`up_volume_confirm_10d`、`up_volume_confirm_20d` — 上涨 + 放量；同花顺看阳线、`VOL` 放大、`AMO` 放大、量比放大。
- `down_volume_risk_5d`、`down_volume_risk_10d`、`down_volume_risk_20d` — 下跌 + 放量；同花顺看大阴线、`VOL` 明显放大、`AMO` 明显放大，偏风险信号。
- `pullback_shrink_volume_5d`、`pullback_shrink_volume_10d`、`pullback_shrink_volume_20d` — 价格回调但成交量缩小；同花顺看阴线变小、`VOL` 低于均量线、成交额萎缩。

典型健康强势形态是“上涨放量、回调缩量、再放量突破”。因此缩量回调不是单纯负面信号，通常需要结合趋势位置判断。

### 4.1.3 量价背离

- `volume_up_price_flat_5d`、`volume_up_price_flat_10d`、`volume_up_price_flat_20d` — 成交额明显放大但价格涨不动；同花顺看放量滞涨、长上影、高位横盘。
- `obv_trend_20d`、`obv_trend_60d` — OBV 趋势；同花顺看 `OBV`、能量潮、OBV 均线方向。
- `obv_divergence_20d`、`obv_divergence_60d` — 股价与 OBV 背离；同花顺看 OBV 是否同步创新高或抗跌。
- `price_volume_corr_10d`、`price_volume_corr_20d`、`price_volume_corr_60d` — 价格收益与成交额/成交量变化的滚动相关性，作为量价同步程度的量化版本。

OBV 类表达更贴近看盘语言，`price_volume_corr_*` 更适合量化检验。智能体解析用户语言时，应优先把“OBV背离”“能量潮走强”“放量滞涨”映射到 OBV 或放量滞涨类因子。

### 4.1.4 突破类

- `volume_breakout_5d`、`volume_breakout_10d`、`volume_breakout_20d`、`volume_breakout_60d` — 今日成交量突破 N 日最大量或 N 日均量；同花顺看 `VOL` 柱子突破均量线、成交量创阶段新高。
- `amount_breakout_20d` — 今日成交额突破 20 日成交额水平；同花顺看成交额突然放大。
- `price_volume_breakout_20d`、`price_volume_breakout_60d` — 价格突破关键均线或前高，同时成交量/成交额放大；同花顺看突破 `MA20`、突破 `MA60`、突破平台、突破阶段新高。

放量突破需要结合位置判断：低位放量突破可能是启动，中位放量突破可能是趋势延续，高位放量突破可能是加速，也可能是阶段高潮。

### 4.1.5 日内强弱/K线结构

- `close_position_strength` — `(收盘价 - 最低价) / (最高价 - 最低价)`，判断收盘位置；同花顺看收盘靠近最高价还是最低价。
- `upper_shadow_ratio` — 上影线比例，衡量上方抛压。
- `lower_shadow_ratio` — 下影线比例，衡量下方承接。
- `real_body_ratio` — K线实体比例，衡量当日方向强度。
- `intraday_strength` — 日内强弱综合指标，可由收盘位置、实体比例、上下影线组合得到。

### 4.2 动量因子

回答：最近涨得强不强？是短线强还是中期强？是自己强，还是相对大盘强？是接近新高，还是刚反弹？

### 4.2.1 阶段涨幅

- `momentum_3d` — 3日涨幅，观察极短线异动；同花顺看 3日涨幅、短线异动。
- `momentum_5d` — 5日涨幅，观察一周强弱；同花顺看 5日涨幅、`ROC5`。
- `momentum_10d` — 10日涨幅，观察短波段强弱；同花顺看 10日涨幅、`ROC10`。
- `momentum_20d` — 20日涨幅，观察月度主线；同花顺看 20日涨幅、`ROC20`。
- `momentum_60d` — 60日涨幅，观察季度主线；同花顺看 60日涨幅、`ROC60`。
- `momentum_120d` — 120日涨幅，观察半年趋势。
- `momentum_250d` — 250日涨幅，观察年度强弱。

代码命名建议统一使用 `momentum_Nd`，在 `description` 或别名中标注同花顺对应 `ROC_N`、阶段涨幅、N日涨幅。

### 4.2.2 ROC / MTM

- `roc_5d`、`roc_10d`、`roc_20d`、`roc_60d` — 变动率指标，本质接近 `momentum_Nd`；同花顺看 `ROC`。
- `mtm_5d`、`mtm_10d`、`mtm_20d` — 动量指标，计算当前价格 - N日前价格；同花顺看 `MTM`。

ETF 横截面比较时优先使用百分比变化，也就是 `momentum_Nd` / `roc_Nd`。`MTM` 更适合作为同花顺别名或单标的观察指标，不建议作为默认横截面打分因子。

### 4.2.3 相对强度

- `relative_strength_5d`、`relative_strength_10d`、`relative_strength_20d`、`relative_strength_60d`、`relative_strength_120d` — 标的 N 日涨幅 - 基准 N 日涨幅；同花顺看 ETF 涨幅 vs 沪深300、创业板指、所属行业或同类 ETF。
- `momentum_rank_5d`、`momentum_rank_10d`、`momentum_rank_20d`、`momentum_rank_60d` — 标的 N 日涨幅在全 ETF 池里的排名百分位；同花顺看 ETF 涨幅排名、行业涨幅排名、板块排名。

相对强度解决“自己涨了但是否真的强”的问题。比如 20 日涨 5%，如果市场平均涨 10%，其实偏弱；如果市场平均跌 5%，反而很强。

### 4.2.4 新高与突破

- `new_high_distance_20d`、`new_high_distance_60d`、`new_high_distance_120d`、`new_high_distance_250d` — 当前价 / N 日最高价 - 1；同花顺看阶段新高、20日新高、60日新高、半年新高、一年新高。
- `new_high_breakout_20d`、`new_high_breakout_60d`、`new_high_breakout_120d` — 是否突破 N 日前高；同花顺看突破前高、创阶段新高。

`new_high_distance_*` 是连续指标，越接近 0 越接近新高；`new_high_breakout_*` 是信号型指标，用于识别强趋势启动或加速。

### 4.2.5 动量质量

- `momentum_acceleration_5_20` — 5日动量 - 20日动量，判断短线是否相对月度节奏加速。
- `momentum_acceleration_10_60` — 10日动量 - 60日动量，判断波段是否相对季度趋势加速。
- `momentum_consistency_20d` — 过去20天上涨天数占比；同花顺看阳线数量、连涨天数、红盘天数。
- `momentum_consistency_60d` — 过去60天上涨天数占比，用于判断中期趋势是否持续。

只看涨幅容易被单日暴涨误导，动量稳定性用于区分“一天冲高”和“持续走强”。

### 4.3 反转因子

回答：跌多了能不能修复？是否出现超卖、下方承接、BOLL 下轨反弹、长下影反转？

### 4.3.1 短期超跌/反弹

- `reversal_3d`、`reversal_5d`、`reversal_10d`、`reversal_20d` — N 日反转，近期跌多后观察是否存在短线修复；同花顺看 3日/5日/10日/20日跌幅、短线超跌。
- `consecutive_down_days` — 连续下跌天数；同花顺看连阴、连续绿盘，用于判断短线杀跌是否过度。
- `gap_down_reversal` — 跳空低开后回升；同花顺看跳空缺口、低开高走。
- `long_lower_shadow_reversal` — 长下影反转；同花顺看下影线、下方承接、探底回升。

### 4.3.2 乖离率/均值回归

- `bias_5d`、`bias_10d`、`bias_20d`、`bias_60d` — 收盘价相对 N 日均线的乖离率；同花顺看 `BIAS`、价格偏离均线程度。
- `ma_deviation_reversal_20d` — 价格偏离20日均线后的均值回归；同花顺看 `BIAS20`、`C / MA20 - 1`。

### 4.3.3 超买超卖指标

- `rsi_6d`、`rsi_12d`、`rsi_24d` — 多周期 RSI；同花顺看 `RSI`、短期/中期超买超卖。
- `kdj_k`、`kdj_d`、`kdj_j` — KDJ 三线；同花顺看 K、D、J 值和金叉死叉。
- `kdj_oversold` — KDJ 低位超卖信号；同花顺看 J 值低位、KDJ 低位金叉。
- `wr_14d` — Williams %R 威廉指标；同花顺看 `WR` 超买超卖。
- `cci_14d` — CCI 顺势指标；同花顺看 `CCI` 超买超卖和趋势拐点。

### 4.3.4 BOLL 反转

- `boll_lower_rebound` — 跌破或触及 BOLL 下轨后回升；同花顺看 BOLL 下轨、跌破下轨后回到轨道内。
- `boll_percent_b` — 收盘价在 BOLL 上下轨之间的位置；同花顺看价格靠近上轨、中轨、下轨。

反转因子必须结合趋势因子使用。比如“跌破多条均线但 RSI/KDJ 超卖”只能说明可能反弹，不代表趋势已经修复。

### 4.4 趋势因子

回答：当前是否处于上升趋势？趋势是否增强？MACD 是否转强？ADX 是否有力度？SAR 是否翻多？BOLL 是否进入强势通道？

### 4.4.1 均线趋势

- `ma_gap_5d`、`ma_gap_10d`、`ma_gap_20d`、`ma_gap_60d`、`ma_gap_120d`、`ma_gap_250d` — 收盘价相对 N 日均线的位置；同花顺看 `MA5`、`MA10`、`MA20`、`MA60`、半年线、年线。
- `ma_slope_20d`、`ma_slope_60d` — 均线斜率，判断趋势是否向上；同花顺看均线方向。
- `trend_strength` — 5日均线相对20日均线的强弱；同花顺看短期均线是否压过中期均线。
- `ma_alignment_score` — 均线多头排列评分；同花顺看 `MA5 > MA10 > MA20 > MA60`。
- `ema_alignment_score` — 指数均线多头排列评分；同花顺看 `EXPMA`。

### 4.4.2 MACD 趋势

- `macd_dif` — MACD DIF 线；同花顺看 `MACD`。
- `macd_dea` — MACD DEA 线。
- `macd_hist` — MACD 柱体。
- `macd_golden_cross` — MACD 金叉。
- `macd_dead_cross` — MACD 死叉。
- `macd_hist_turn_positive` — MACD 柱体由负转正，表示趋势动能转强。

### 4.4.3 趋势强度指标

- `dmi_adx` — ADX 趋势强度；同花顺看 `DMI / ADX`。
- `dmi_pdi_mdi_diff` — PDI 与 MDI 差值，判断多空方向强弱。
- `sar_trend` — SAR 抛物线趋势状态；同花顺看 `SAR` 翻多/翻空。
- `bbi_trend` — BBI 多空指标趋势；同花顺看 `BBI`。
- `trix_trend` — TRIX 中期趋势；同花顺看 `TRIX`。
- `dma_trend` — DMA 均线差趋势；同花顺看 `DMA`。

### 4.4.4 通道与突破

- `boll_mid_trend` — BOLL 中轨趋势；同花顺看布林中轨方向。
- `boll_upper_breakout` — 放量突破 BOLL 上轨，判断强势突破或短期过热。
- `breakout_60d` — 突破60日区间高点；同花顺看 60日新高、平台突破、`HHV(C,60)`。
- `trend_persistence_20d`、`trend_persistence_60d` — 趋势持续性，统计 N 日内价格位于关键均线上方的比例。

### 4.5 估值因子

回答：行业 ETF 对应资产贵不贵？

- `industry_pe_percentile` — 行业市盈率所处历史分位，越低越便宜；同花顺常看行业 `PE`、市盈率分位
- `industry_pb_percentile` — 行业市净率所处历史分位，越低越便宜；同花顺常看行业 `PB`、市净率分位

这类因子需要行业指数估值或 ETF 成分股加权估值数据，第一版不作为默认因子。

### 4.6 质量因子

回答：ETF 成分股整体基本面质量好不好？

- `weighted_roe` — ETF 成分股按权重加权后的净资产收益率；同花顺常看 `ROE`、净资产收益率

这类因子需要 ETF 成分股权重和个股财务数据，第一版不作为默认因子。

### 4.7 成长因子

回答：ETF 成分股或跟踪行业是否具备增长能力？

- `weighted_profit_growth` — ETF 成分股按权重加权后的利润增速；同花顺常看净利润增长率、归母净利润同比

这类因子需要 ETF 成分股权重和个股增长数据，第一版不作为默认因子。

### 4.8 风险因子

回答：这个 ETF 风险大不大？是否高位过热？是否短线涨太急？是否成交额拥挤？是否流动性不足？

- `liquidity_20d` — 20日平均成交额，衡量买卖是否方便；同花顺常看 `AMO`、20日均成交额、流动性。
- `liquidity_risk` — 流动性不足风险，通常由成交额过低、成交不连续、规模过小等条件组合。
- `volatility_5d`、`volatility_10d`、`volatility_20d`、`volatility_60d` — 多周期收益波动率；同花顺看波动率、标准差。
- `downside_volatility_20d` — 只统计下跌日的20日波动，衡量下行风险。
- `beta_60d` — 相对基准指数的60日贝塔，衡量跟随大盘波动程度；同花顺看 `BETA`。
- `risk_adjusted_momentum_20d` — 20日动量 / 20日波动率，看单位风险下的趋势收益。
- `max_drawdown_20d`、`max_drawdown_60d` — N 日最大回撤。
- `drawdown_from_high_20d`、`drawdown_from_high_60d` — 当前价格相对 N 日高点的回撤幅度。
- `distance_from_ma20_overheat`、`distance_from_ma60_overheat` — 距离 MA20/MA60 过远，判断短期或中期过热。
- `turnover_overheat_5d` — 换手率短期异常放大。
- `amount_overheat_5d` — 成交额短期异常放大。
- `rsi_overheat` — RSI 高位过热。
- `boll_upper_overheat` — 价格贴近或突破 BOLL 上轨后的过热风险。
- `gap_risk` — 跳空缺口风险。
- `single_day_large_drop` — 单日大跌风险。
- `single_day_large_rise` — 单日大涨后的短期过热风险。
- `consecutive_large_volume` — 连续放量，判断资金拥挤或分歧放大。
- `crowding_amount_ratio_5d` — 5日成交额拥挤度，判断短期资金是否过热。
- `short_term_overheat_5d` — 短期涨幅和放量同时过高，判断是否过热。

风险因子不只是为了筛掉“差的标的”，更重要的是避免在过热位置追入。ETF 轮动里，很多标的不是不能买，而是不能在拥挤和过热区买。

### 4.9 ETF 专属因子

回答：这个 ETF 本身是否适合交易？有没有折溢价、规模、跟踪误差、流动性、成分股集中度等 ETF 特有风险？

- `etf_premium_discount` — ETF 折溢价率，判断是否相对净值买贵或买便宜；同花顺看 IOPV 折溢价。
- `iopv_deviation` — 交易价格相对 IOPV 的偏离程度。
- `fund_size` — 基金规模，规模过小可能有流动性和清盘风险。
- `tracking_error` — 跟踪误差，判断 ETF 是否稳定跟踪标的指数。
- `bid_ask_spread` — 买卖价差，衡量交易成本。
- `turnover_amount_20d` — 20日平均成交额，ETF 交易可进出能力。
- `liquidity_score` — ETF 流动性综合分，可由成交额、成交量、买卖价差、规模综合得到。
- `top10_weight_concentration` — 前十大持仓集中度，判断是否被少数龙头绑架。
- `industry_concentration` — 行业集中度，判断 ETF 暴露是否过于集中。
- `constituent_momentum` — 成分股加权动量，判断 ETF 内部成分是否一致走强。
- `constituent_volatility` — 成分股加权波动率。
- `constituent_valuation_pe` — 成分股加权 PE。
- `constituent_valuation_pb` — 成分股加权 PB。

ETF 专属因子需要额外字段支持。第一版可以先落地 `fund_size`、`turnover_amount_20d`、`iopv_deviation`，成分股类因子后续再接成分股权重和财务数据。

### 4.10 资金流/主力因子

回答：是否有主力资金、大单资金或机构资金参与？资金方向是流入、流出，还是分歧扩大？

- `main_money_inflow` — 主力资金净流入；同花顺看主力资金流向；`data_dependency = platform_field`，`bigquant_ready = Depends`。
- `main_buy_sell_strength` — 主力买卖强弱；同花顺看主力买卖；`data_dependency = platform_field`，`bigquant_ready = Depends`。
- `large_order_net_volume` — 大单净量；同花顺看大单净量、DDX、DDE；`data_dependency = platform_field`，`bigquant_ready = Depends`。
- `large_order_net_amount` — 大单净额；同花顺看大单金额；`data_dependency = platform_field`，`bigquant_ready = Depends`。
- `bbd_net_inflow` — BBD 大单资金净差；`data_dependency = platform_field`，`bigquant_ready = Depends`。
- `institution_activity` — 机构活跃度；`data_dependency = platform_field` 或 `external_data`，`bigquant_ready = Depends`。
- `institution_research_heat` — 机构研究度或研报关注度；`data_dependency = external_data`，`bigquant_ready = False`。
- `main_crowding_score` — 主力抱团度或资金拥挤度；`data_dependency = platform_field`，`bigquant_ready = Depends`。

资金流/主力因子需要区分数据依赖：`OBV`、`VR`、成交额、成交量、量比、换手率可以由 OHLCV 或基础行情计算；主力资金流向、大单净量、BBD、DDX、DDE、机构活跃度等通常需要平台专有字段或外部数据支持。

### 4.11 市场环境因子

回答：当前市场适不适合进攻？是趋势市、震荡市，还是风险偏好下降？ETF 轮动是否需要降低仓位或转向防守方向？

- `index_momentum_20d` — 基准指数20日涨幅，判断大盘短中期方向。
- `index_trend_strength` — 基准指数趋势强度，例如 `MA5 / MA20 - 1` 或 `close / MA60 - 1`。
- `market_breadth` — 市场宽度，例如上涨标的数量 / 全市场标的数量。
- `up_down_ratio` — 上涨家数 / 下跌家数。
- `limit_up_count` — 涨停数量，衡量市场进攻强度。
- `limit_down_count` — 跌停数量，衡量市场风险释放。
- `northbound_flow` — 北向资金净流入，若数据可得可作为风险偏好指标。
- `market_volatility` — 大盘波动率，衡量市场不确定性。
- `risk_on_score` — 市场风险偏好综合分，可由指数动量、市场宽度、涨跌家数、波动率、资金流综合得到。

市场环境因子用于控制总仓位和风格选择。市场环境较差时，即使某个 ETF 个体因子分高，也应降低仓位、提高风险过滤阈值，或转向低波动、防守型 ETF。

## 5. 因子元数据规范

后续正式因子库建议每个因子记录以下元数据，方便智能体检索、解释、降级和组装研究计划：

- `name`：因子英文名。
- `ths_name`：同花顺名称。
- `aliases`：同花顺别名和用户自然语言说法，例如“OBV上涨”“能量潮走强”“缩量回调”。
- `description`：中文解释。
- `formula`：可计算公式或伪公式。
- `direction`：越高越好、越低越好或双向解释。
- `category`：量价、动量、反转、趋势、风险、ETF专属、资金流等。
- `is_raw_indicator`：是否原生指标。
- `is_derived_factor`：是否加工因子。
- `is_composite_score`：是否综合评分。
- `asset_scope`：适用 ETF、个股、指数或行业。
- `required_columns`：所需字段。
- `default_period`：默认周期。
- `optional_periods`：可选周期。
- `data_dependency`：`ohlcv`、`platform_field`、`fundamental`、`etf_specific`、`constituent_data`。
- `bigquant_ready`：当前 BigQuant 环境是否可直接计算。
- `uses_future_data`：是否使用未来数据，特征必须为 `False`。
- `research_usage`：适合条件研究、因子打分、风险过滤还是报告解释。
- `is_redundant_with`：容易重复的相关因子组。
- `enter_final_score`：是否进入最终综合打分。
- `default_weight`：默认权重。
- `missing_value_handling`：缺失值处理。
- `outlier_handling`：极值处理。
- `standardization`：标准化方式。

## 6. 可计算公式与重复因子组

### 6.1 示例公式

以下先记录容易落地的计算口径，后续同步到 `src/etf_factor_library.py` 时应写入 `formula` 或 `pandas_hint`。

| 因子 | 示例公式/规则 | 说明 |
| --- | --- | --- |
| `momentum_consistency_20d` | `过去20日 return_1d > 0 的天数 / 20` | 上涨天数占比，衡量动量稳定性 |
| `trend_persistence_20d` | `过去20日 close > MA(close, 20) 的天数 / 20` | 位于均线上方的比例，衡量趋势持续性 |
| `short_term_overheat_5d` | `momentum_5d 分位数高 + amount_ratio_5d 高 + rsi_6d 高` | 短线涨幅、成交额、超买共同过热 |
| `liquidity_risk` | `amount_20d 过低 或 fund_size 过小 或 bid_ask_spread 过高` | ETF 流动性风险 |
| `boll_lower_rebound` | `close 昨日低于 BOLL 下轨 且 今日 close 回到下轨上方` | BOLL 下轨反弹 |
| `boll_percent_b` | `(close - boll_lower) / (boll_upper - boll_lower)` | 价格在布林通道中的位置 |
| `main_crowding_score` | `main_money_inflow 分位数 + large_order_net_amount 分位数 + turnover_overheat` | 依赖平台资金流字段 |
| `obv_divergence_20d` | `price 创20日新高但 OBV 未创新高，或 price 横盘但 OBV 上升` | OBV 背离规则可拆成多个信号 |
| `price_volume_breakout_20d` | `close > REF(HHV(close, 20), 1) 且 amount_ratio_20d > 1` | 价格放量突破前高 |
| `risk_on_score` | `index_momentum_20d + market_breadth - market_volatility` | 市场环境综合分，实际需标准化后合成 |

### 6.2 重复因子组

以下因子含义高度相关，最终打分时不能简单重复加权。正式元数据中应写入 `is_redundant_with`。

| 重复组 | 相关因子 | 处理建议 |
| --- | --- | --- |
| 20日动量组 | `momentum_20d`、`roc_20d`、20日涨幅、`C / REF(C,20) - 1` | 本质相同，默认只保留 `momentum_20d` |
| 动量相对强弱组 | `momentum_20d`、`momentum_rank_20d`、`relative_strength_20d` | 都偏动量暴露，可同时研究，但打分需降权 |
| 成交额活跃组 | `amount_ratio_20d`、`amount_breakout_20d`、`amount_overheat_5d`、`crowding_amount_ratio_5d` | 区分“正常活跃”和“过热拥挤” |
| BOLL位置组 | `boll_percent_b`、`boll_lower_rebound`、`boll_upper_overheat` | 一个是位置，一个是反转，一个是过热风险 |
| 均线趋势组 | `ma_gap_20d`、`trend_strength`、`trend_persistence_20d`、`ma_alignment_score` | 可合成趋势分，不宜全部独立等权 |
| 资金流组 | `main_money_inflow`、`large_order_net_amount`、`bbd_net_inflow`、`main_crowding_score` | 依赖同类平台字段，需确认字段可用性 |

## 7. 使用原则

第一版不要一次性使用全部因子。建议流程：

```text
先使用 MVP 默认 5 个因子
→ 做单因子 IC / 分层收益
→ 剔除不稳定因子
→ 再尝试加入 2-3 个扩展因子
→ 比较等权、假设权重、ICIR 权重
```

任何因子都不能使用未来数据。`future_return_5d`、`future_return_20d` 只能作为标签，不能参与评分和选 ETF。
