# AutoETF Research Agent 方案说明

## 1. 项目定位

AutoETF Research Agent 是一个面向行业 ETF / 主题 ETF 轮动研究的量化研究辅助系统，基于 BigQuant 平台数据 + DeepSeek LLM 驱动。支持两种研究模式：

- **条件事件研究**：检验 ETF 在满足特定条件（如量比 > 1、换手率 < 5、今日上涨）后，下一交易日上涨概率是否显著高于全样本基准
- **多因子研究**：自然语言解析 → 因子生成 → IC 分析 → 复合打分 → Top% 等权轮动回测 → 诊断 → 报告

仅用于历史回测、统计检验和策略研究，不构成任何投资建议。

## 2. 核心流程

```
用户自然语言问题
  │
  ├─ [条件事件研究]
  │   → 条件解析（condition_parser.py / expression_knowledge_base.py）
  │   → BigQuant 数据获取（data_loader_bigquant.py）
  │   → 条件概率检验（condition_analysis.py）
  │   → 分年份稳定性
  │   → 条件有效性诊断
  │   → Markdown 研究报告
  │
  └─ [多因子研究]
      → 因子意图解析（factor_resolver.py / expression_knowledge_base.py）
      → 因子候选生成（factor_generator.py / etf_factor_library.py）
      → BigQuant 数据获取（data_loader_bigquant.py）
      → IC 分析 / 分位数收益（factor_analysis.py）
      → 权重方案构建（scoring.py）
      → 因子归一化 + 复合打分（scoring.py）
      → Top% 等权轮动回测（backtest.py）
      → 策略诊断（diagnosis.py）
      → Markdown 研究报告 + 图表（report.py / plotting.py）
```

## 4. 现有功能

### 4.1 条件事件研究（完整可用）

| 模块 | 功能 | 文件 |
|---|---|---|
| 条件解析 | NL → 结构化条件假设 | `condition_parser.py` |
| 条件概率检验 | P(上涨\|条件) vs P(上涨\|基准)，概率提升、均值提升 | `condition_analysis.py` |
| 分年份稳定性 | 按年统计概率稳定性 | `condition_analysis.py` |
| 条件诊断 | 有效性评价 + 研究决策（继续/放弃/需改进） | `condition_analysis.py` |
| 报告生成 | Markdown 报告 + 保存 | `condition_report.py` |

### 4.2 多因子研究（完整可用）

| 模块 | 功能 | 文件 |
|---|---|---|
| 因子意图解析 | NL → 因子/条件/目标、方向、类别 | `factor_resolver.py` |
| 因子候选生成 | 按类别/方向/相关性选因子，校验未来函数 | `factor_generator.py`, `etf_factor_library.py` |
| 因子可用性过滤 | 按数据库列名过滤可用因子 | `factor_availability.py` |
| IC 分析 | 日度 IC、均值 IC、ICIR、IC 累积、IC 系列 | `factor_analysis.py` |
| 分位数收益 | Top/Bottom 分组收益、多空收益差 | `factor_analysis.py` |
| 因子分解 | 因子分解 + 正交化 | `factor_analysis.py` |
| 胜率/优势比 | 正收益比率 + 胜率分析 | `factor_analysis.py` |
| 归一化 | 线性/范围型/U 型/倒 U 型归一化 | `scoring.py` |
| 复合打分 | 加权复合 + metadata 驱动打分 | `scoring.py` |
| 权重方案 | IC 加权、等权、ICIR 加权、衰减加权、假设加权 | `scoring.py` |
| 回测 | Top% 等权轮动、交易成本、绩效统计 | `backtest.py` |
| 策略诊断 | 因子 IC、权重、回测表现综合诊断 | `diagnosis.py` |
| 报告 | 结构化 Markdown 报告 | `report.py` |
| 图表 | 累计收益、回撤、IC 系列、分位数收益图 | `plotting.py` |

### 4.3 表达式知识库（4 层映射）

| 层级 | 说明 |
|---|---|
| raw_data | 原始数据列（close, volume, amount, turn 等） |
| native_indicator | 平台原生指标（量比、换手率、涨跌幅等） |
| processed_factor | 加工因子（动量、趋势强度、波动率等） |
| composite_score | 复合评分指标 |

### 4.4 智能体聊天（完整可用）

| 功能 | 说明 |
|---|---|
| LLM 路由 | DeepSeek 判断用户意图 → 选择条件研究或因子研究 pipeline |
| 规则兜底 | LLM 失败时回退到 rule-based 判断 |
| 工具注册 | 13 个工具统一注册 + `call_agent_tool()` 分发 |
| 对话状态管理 | 上下文跟踪、连续追问、对话历史 |

| 工具输出摘要 | LLM 将结构化结果转成中文自然语言摘要 |

## 5. BigQuant 数据要求

所有 BigQuant 相关逻辑集中在 `data_probe_bigquant.py` 和 `data_loader_bigquant.py`。

取数前建议先运行字段探测确认可用列：

```python
from src.data_probe_bigquant import run_data_probe
probe_results = run_data_probe()
```

当前 BigQuant 表及关键列：

| 表名 | 关键列 |
|---|---|
| `cn_fund_bar1d` | `close`, `volume`, `amount`, `turn`（换手率）, `open`, `high`, `low`, `return` |
| `cn_stock_index_bar1d` | 基准指数行情 |
| `cn_fund_real_bar1d` | 实时/附加行情 |
| `cn_fund_nav` | 净值数据 |
| `cn_fund_static_data` | 基金静态信息 |

## 6. 未来函数约束

- 特征因子只能用当前及历史数据，禁止 `m_lead`、`future_*` 前缀
- `future_return_5d`、`future_return_20d` 仅作评价标签，不参与排序/评分/选 ETF

## 7. 回测说明

- 每个调仓日计算复合得分 → 选 Top% ETF → 等权持有
- 扣除简化交易成本（bps）
- 统计：累计收益、年化收益、最大回撤、夏普比率、胜率、换手率
- 不包含：真实撮合、滑点、折溢价、流动性冲击
